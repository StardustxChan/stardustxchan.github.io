macOS, GnomeLinux 桌面设置一览

思想 P5 化.

硬件

最近在虚拟机下摸鱼的时间越来越长了, Retina MacBook Pro15.2017 其实用虚拟机的性能还算可以, 但是, 室温 20度 就能保持 60-70度 的高温, 还是挺难受, 风扇也是日常最低档 2500rpm 养老级别转速.

待机时, 渴望蓝天模具的可设置停转风扇.

macOS

自从 2015 年接触系统以来, 就很少用桌面, 一般就在层层叠叠的窗口里相互切换.
Person5 狂热者的信仰壁纸.

菜单栏

因为上菜单栏和 Dock 会透射壁纸的颜色, 即便像我这样不看桌面, 也设置了 “定时随机播放图片” 换换菜单栏颜色

菜单栏有亮色和暗色两种可选, 个人虽然果断得优先可读和舒适感, 把所有编辑器和 IDE , 甚至 Chrome 网页的背景色设置成暗色. 但是菜单栏在设置成暗色后, 即便背景色在透明度和模糊度方面完美, 但是作为前景色的白色对比度也太高了, 个人觉得很刺眼, 相当刺眼.

Dock

如果是黑苹果或者公用机器, 关闭 SIP(系统完整性验证), 再使用 cDock 对 Dock 样式进行为所欲为的改动也没事.
以前的设置如下保留了透明度, 只更改了 Dock 底色为粉色和紫色.

图标

  1. 用 LiteIcon 集成管理和替换图片到 icon 即可.
  2. 开启 SIP 的系统无法替换系统程序, 在这里我关闭, 替换 Finder 后又开启.
  3. 当然, 如果升级系统, 系统程序的图标都会还原.
  4. 如图我仅替换了:
    1. Finder: 以大版本默认桌面为壁纸的 MockUp, 个人的趣味
    2. PDF Expert: 不喜欢原版重阴影的拟物化
    3. OmniFocus: 如上扁平化
    4. QQ: 个人的恶趣味
    5. 网易云音乐: 全平台的圆角矩形有些腻了, 惯例在 Mac 端的软件都是要去框的
    6. Parallel Desktop: 用各机型对应的图标

Linux

功能 UI

  1. 常用软件不多, 因此把 Dock 放到侧边也不会出现 macOS 那样过多软件挤在一起, 图标过小的情况
  2. 个人明确感到 Gnome 下的默认上栏在功能上可有可无, 倾向直接隐藏
  3. 窗口的 “关闭-最大化-最小化” 当然是遵从习惯放到左边

主题和图标 UI

  1. GTK 和 Icons 都是用的刚摸上 Gnome 时 B站找的教程设置的
  2. 就 Material Design 来说, 这套主题算是满足并且白搭目前的扁平化风格了
  3. Dock 没有 macOS 的模糊浑身难受, 只能设置成低透明度或不透明了.

P5 天下第一

讲起使用逻辑来一套一套的, 看着易用而现代化的 UI, 心里还不是想着 “P5 天下第一”

为了易用性和可读性放弃 “土嗨” 的界面, 却还是会对 P5 强烈的风格倾心.

“啊, 这动画, 这配色, 这字体”

似乎只能自己动手做界面了

总体方向不会和游戏 UI 颜色一样啊, 高对比的黑色和红色日常太晃眼了. 当然会综合背景模糊和透明度.

挖了坑也不知道会不会填.

Dock 肯定会保留

菜单风格

  1. 目前构想是把在最前的 App 搞成游戏菜单里选中的动画, 然而似乎会晃眼和看腻
  2. 打开却不在最前的也许可以只留一个底色

Dock 底色黑, 打开的底色白, 最前的白底色之上有选中动画
Dock 形状不规则

Palace 风格

  1. 由不规则形状, 第一个想到是泼墨, 自然也联想到 Palace 跑步的特效
  2. 因此三种情况可以互换形状: 规则形状Dock, 泼墨底色

然而, 泼墨作为鼠标停留和点击也是极好的

通知栏

自然是选择游戏中的指令样式, 然而我并没有消息 :(
